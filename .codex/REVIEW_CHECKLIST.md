# Completion Checklist

Before marking a ticket READY_FOR_REVIEW or DONE, verify:

- [ ] root cause solved
- [ ] no hacky shortcuts
- [ ] no placeholder or fake implementation
- [ ] no guessed context
- [ ] acceptance criteria satisfied
- [ ] validation commands executed or blocker precisely documented
- [ ] touched code is cleaner or at least not worse
- [ ] architecture respected or improved
- [ ] dead code removed where relevant
- [ ] final summary written
- [ ] residual risks documented honestly
